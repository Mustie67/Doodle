# Doodle Chalk AI

An AI doodle-guessing app: draw on a chalkboard-styled canvas and a small CNN predicts what it is in real time (top-3 guesses shown).

Project layout
- data/                     → Quick, Draw! .npy bitmap files (28×28, one per category)
- model/
  - train_model.py          → load data, build CNN, train, evaluate, save
  - doodle_cnn.h5           → trained model (generated by the script)
  - metrics/                → confusion_matrix.png, metrics.json, training_curves.png
- app.py                    → Streamlit app (chalkboard UI)
- style.css                 → injected custom CSS for the chalkboard look
- requirements.txt
- data_prep.py              → dataset download / synthetic-fallback helper

Categories (10, fixed)
- cat, house, tree, car, fish, star, umbrella, banana, bicycle, clock

Quick start (using synthetic data)
1. Create a virtualenv and install dependencies:
   pip install -r requirements.txt

2. Generate synthetic data (so you can run everything inside a sandbox):
   python data_prep.py
   This writes .npy files into data/ by default (USE_SYNTHETIC = True in data_prep.py).

3. Train:
   python model/train_model.py
   Outputs saved to model/ and model/metrics/.

4. Launch the Streamlit app:
   streamlit run app.py

Using the real Quick, Draw! dataset
- If you prefer the real dataset, download each category's numpy bitmap file (.npy) from Google's Quick, Draw! dataset:
  Example curl command (run on your machine where storage.googleapis.com is reachable):
    mkdir -p data
    curl -L -o data/cat.npy https://storage.googleapis.com/quickdraw_dataset/full/numpy_bitmap/cat.npy
    curl -L -o data/house.npy https://storage.googleapis.com/quickdraw_dataset/full/numpy_bitmap/house.npy
    ...repeat for each category listed above...

- Or use the Python snippet in data_prep.py to fetch them.

Notes on preprocessing and model
- Input: 28×28 grayscale bitmaps normalized to [0,1], reshaped to (28,28,1)
- Labels: one-hot encoded
- Split: stratified train/test 80/20
- Model: small CNN (Conv2D -> MaxPool -> Conv2D -> MaxPool -> Dense -> Dropout -> Dense softmax)
- Training outputs: model/doodle_cnn.h5, model/metrics/metrics.json, confusion matrix and training curves plots.

Design & UI
- Chalkboard palette and hand-drawn fonts (Caveat for title, Space Grotesk for body).
- Predictions are updated after each stroke (mouse-up). The "Erase" button performs a simple wipe animation.
- Empty canvas shows a friendly chalk prompt.

Troubleshooting
- If model loading fails in the app: ensure model/doodle_cnn.h5 exists (created by training script).
- If you can't reach storage.googleapis.com from your environment, use synthetic mode (default) to demo the pipeline.
